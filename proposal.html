一、项目名称和 GitHub 仓库地址

项目名称：MoonKit（MoonBit 生态扩展工具库）
GitHub 仓库：https://github.com/bzhangui/moonkit
开源协议：Apache-2.0

二、项目简介

MoonKit 是一个面向 MoonBit 语言的通用工具库，目前提供六个子模块：toml（TOML 配置解析）、semver（语义化版本处理）、validators（数据校验）、strcase（字符串命名格式转换）、ulid（ULID 唯一标识符生成）和 convert（数据类型转换）。共计约 4000 行 MoonBit 代码、23 个源文件、10 个测试文件、22 次 Git 提交。

MoonBit 语言核心和标准库发展迅速，但实用工具库层面仍有空白，开发者常需从零实现配置解析、输入校验、版本比较等基础功能。MoonKit 试图填补这些缺口，为 MoonBit 开发者提供一套开箱即用、零外部依赖、可独立引用的基础组件库，降低日常开发中的重复劳动。

三、项目方向与适用场景

项目方向：原创工具库。

适用场景及对应模块说明如下：

适用场景对应模块具体功能
应用程序配置管理tomlTOML v1.0 类型系统、文档序列化器、Builder 模式链式构造器；可读写 moon.mod 等标准配置文件，适用于 CLI 工具和 Web 应用配置
包管理与版本控制semverSemVer 2.0 解析器、Precedence 比较器、版本范围匹配（支持 ^ ~ >= <= > < = 运算符）；可用于依赖解析、API 版本兼容性检查
用户输入校验validators常见格式校验（邮箱、URL、非空、长度范围、正则匹配），以及可组合的 Schema 链式校验框架，适用于 Web API 和表单验证
代码生成与数据映射strcase五种命名风格双向转换（camelCase、PascalCase、snake_case、kebab-case、Title Case），自动识别分隔符
分布式 ID 生成ulidULID 规范生成器（时间戳 + 随机数）、Base32 Crockford 编解码、合法性校验，适合日志追踪和消息队列
基础类型转换convert字符串与数值互转、十六进制编解码、格式化填充、类型判断等辅助工具

四、拟实现的核心功能

以上六个模块的全部功能已实现完毕，项目当前状态：

  • 代码量：3994 LOC（不含注释和空行），满足大赛 4k~10k 的规模要求
  • 测试：每个模块配备独立测试文件，共 10 个测试文件，覆盖正常路径、边界条件和错误分支。使用 moon test 可执行全部测试
  • 架构：主包 moonkit 统一导出六个子包，也可按需单独引用（如 @moonkit/semver）。所有可能失败的函数统一使用 Result[T, String] 返回值，不抛异常
  • 构建：首选编译目标 wasm-gc，适配 WebAssembly 场景；已配置 GitHub Actions CI 工作流，确保持续可构建
  • 依赖：仅依赖 moonbitlang/core 标准库，无任何第三方依赖

五、是否为原创项目

本项目为原创项目。所有代码由本人独立设计与编写，未移植或复制任何第三方开源项目的源代码。

部分模块在 API 设计层面参考了 Rust 等语言成熟生态库的命名习惯和组织方式（如 TOML 模块参考了 toml crate 和 serde 序列化器的设计思路，semver 模块参考了 semver crate 的版本号比较逻辑），但所有实现代码均为在 MoonBit 语言下独立编写，与参考项目无代码层面的直接关联。
